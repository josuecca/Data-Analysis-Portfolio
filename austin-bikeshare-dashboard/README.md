Austin Bikeshare Operational Dashboard (2019)
Project Overview

This project analyzes monthly bike usage patterns in the Austin Bikeshare system during 2019.
The goal was to identify seasonal demand trends and operational insights using SQL and Google Sheets.

This dashboard complements my SQL analysis by translating aggregated data into executive-level visual insights.

Business Objective

To evaluate:

  - Total trips per month

  - Average trip duration per month

  - Seasonal demand fluctuations

  - Peak and lowest activity periods

Understanding these patterns can help optimize bike distribution, maintenance planning, and operational decision-making.

Dataset

Source:
bigquery-public-data.austin_bikeshare.bikeshare_trips

The data was:

  1. Queried using BigQuery SQL

  2. Filtered to the 2019 calendar year

  3. Aggregated by month

  4. Exported as CSV

  5. Imported into Google Sheets for dashboard creation

Key Metrics (2019)

  - Total Trips: 106,540

  - Average Trip Duration: 49.7 minutes

  - Peak Month: March (15,431 trips)

  - Lowest Activity Month: December (4,538 trips)

Key Insights

  - Bike usage shows strong seasonality, with peak demand occurring in March.

  - Demand declines significantly in December, indicating weather-related or seasonal behavioral patterns.

  - Although total trips decrease in late summer, average trip duration increases — with August showing the highest average duration (~68 minutes).

  - Seasonal fluctuations suggest the need for dynamic operational planning and maintenance scheduling.

Dashboard Components

  - KPI summary section (Total Trips, Avg Duration, Peak Month, Lowest Month)

  - Line chart: Monthly Total Trips

  - Bar chart: Monthly Average Duration

Tools Used

  - Google BigQuery (SQL)

  - Google Sheets (Dashboard & KPI calculations)
